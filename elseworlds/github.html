Adding Existing Directories from Personal Computer to GitHub Using Terminal

GitHub is a popular platform used for version control and code management. It allows you to store and share your code with others easily. This guide will walk you through the steps of adding your existing directories to GitHub using the terminal.

Prerequisites

Before starting, make sure you have the following:

  • A GitHub account
  • Git installed on your computer
  • A directory on your computer that you want to add to GitHub

Steps

1. Navigate to the directory you want to add to GitHub using the terminal.

cd /path/to/directory

2. Initialize Git in the directory.

git init

3. Create a new repository on GitHub.

  • Log in to your GitHub account and navigate to the repository creation page.
  • Give your repository a name and a description.
  • Choose whether the repository should be public or private.
  • Do not select the option to initialize the repository with a README file, since you already have existing files to add.

4. Add the remote repository as the origin.

git remote add origin https://github.com/username/repository.git

Replace username with your GitHub username and repository with the name of the repository you just created.

5. Add all the files to the staging area.

git add --all

6. Commit the changes.

git commit -m "First commit"

7. Choose the branch.

Here I chose the branch named “main”; some people might use “master”.

git branch -M main

8. Push the changes to GitHub.

git push -u origin main

Note: If your branch name is “master” or something else, change “main” to your branch name.

Un-track Files on Git

If you want to un-track files on Git, use the following command:

git rm --cached filename
